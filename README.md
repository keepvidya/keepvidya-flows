# Keepvidya Flows

**Local-first AI Flow apps for your desktop.** Something boring in — a real thing you own out, running on your machine.

Drop in a dull PDF, a text file, or a web link, and Flows turns it into an **illustrated storybook** or a **playable game** — generated by a bundled local model that never leaves your computer, or by your own cloud key (BYOK). Everything you make is saved to a library that's yours to keep.

> Brand & UX are shared with [keepvidya.com](https://keepvidya.com): Source Serif 4 + Source Sans 3, ink + copper, light/dark.

## What it does

1. **Bring something in** — paste text, upload a file (PDF / DOCX / HTML / RTF / TXT / MD / CSV / JSON), or give it a web link.
2. **Pick what to make** — an illustrated storybook (a PDF you read and download) or a self-contained, playable HTML game.
3. **Run it on your machine** — a bundled [Shiva](https://keepvidya.com/products/core/) model, fully offline; or bring your own key for a bigger cloud model.
4. **Keep it** — every result auto-saves to your **Library**, where you can re-open, **Novelize** (expand a storybook into a richer book), or delete.

### Highlights

- **Truly local** — the default engine talks to a local [Ollama](https://ollama.com) running a Shiva model. Nothing is sent anywhere.
- **BYOK, same set as [Knovex](https://keepvidya.com/products/knovex/)** — OpenAI, Anthropic, Groq, Google Gemini, Cerebras, AWS Bedrock, and Ollama. Keys are encrypted on-device with Electron `safeStorage`.
- **Domain-agnostic** — it adapts to whatever the document is *about* (nutrition, finance, history, space, science, tech…), with illustration scenes routed across ~11 domains.
- **Real document loading** — a small, framework-style multi-format loader (the Node analogue of LangChain's loaders): PDF via PDF.js/`pdf-parse`, DOCX via `mammoth`, HTML/RTF strippers, everything else as UTF-8 — never raw bytes.
- **Web links over the internet** — fetches a page (or a remote PDF), strips it to text, and feeds that to the model. **SSRF-guarded**: only public http/https hosts; private/loopback/link-local/reserved IPs and the cloud-metadata address are blocked, hostnames are DNS-resolved and re-checked, and every redirect hop is re-validated.
- **Entertaining installer** — pick *Local & private* (bundled model, offline) or *Connect a key* (BYOK), with a system check that detects existing Ollama, sizes the model to your hardware, and a Catch-the-Spark mini-game while it installs.
- **Frameless, fully-branded** — custom window chrome, taskbar/Start-menu icon, light/dark.

## Run from source

```bash
cd app
npm install
npm start            # launch the Electron app   (npm run dev for --dev)
```

The default **Local** engine needs a local Ollama with a `shiva-chat` model; otherwise add a BYOK key in Settings.

## Build installers

```bash
cd app
npm run build:win    # or build:mac / build:linux  → app/release/
```

Releases are produced by CI: push a tag `vX.Y.Z` and [`.github/workflows/package.yml`](.github/workflows/package.yml) builds installers + updater metadata for all three platforms and publishes them to the GitHub Release via `electron-builder --publish always`.

## How it works

Node-only — there is no separate backend. Flows are LLM calls, so the engine runs in-process in Electron's main and talks to local Ollama and BYOK providers over HTTP.

| Module | Role |
|--------|------|
| `app/main.js` | Electron main: frameless window, tray, IPC, auto-updater, file/web intake. |
| `app/preload.js` | `contextBridge` → `window.kvflows.*` (no Node in the renderer). |
| `app/lib/flowEngine.js` | The `SPECIALISTS` registry — *one base per domain, many prompt-hats*. Storybook + Novelize are **one model in two modes**; game → a Coder base (falls back to built-in Snake); extract → a chat model. |
| `app/lib/providers.js` | Real HTTP calls for the 7 BYOK providers (+ Ollama). |
| `app/lib/docloader.js` | Multi-format document loader + SSRF-guarded URL fetch. |
| `app/lib/library.js` | File-based storage of everything you make (`userData/library/`). |
| `app/lib/system.js` | First-run probe: detects Ollama, reads hardware, recommends a model. |
| `app/lib/store.js` | Config + keys, encrypted at rest with `safeStorage`. |
| `app/renderer/` | The UI. |

### Models

The local engine runs **Shiva** specialists tuned by Keepvidya — Shiva-Writer (storybook + novelize), Shiva-Chat (extract), Shiva-Code (games). The installer can size the recommendation to your machine (Fast vs Quality). With BYOK you can point any flow at a bigger cloud model.

## Tests

[`TEST-PLAN.md`](TEST-PLAN.md) documents every button and its expected result.

```bash
cd app
node test/make-fixtures.js                # 9 fixtures across txt/md/csv/json/pdf/docx/html, multiple domains
node test/check-extract.js                # every fixture extracts to real text via the document loader
node test/check-domains.js                # scene routing for 11 domains + cross-domain generation (needs Ollama)
node test/check-ssrf.js                   # Web-link SSRF guard: blocks localhost/private/metadata, allows public
node_modules/.bin/electron test/e2e.js    # full UI E2E: installer → app → file + web intake → generation → results
```

Current status: **E2E 43/43**, SSRF 11/11, domains 11/11.
